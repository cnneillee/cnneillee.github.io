一、AAPT err(Facade for ……): libpng error: Not a PNG file.问题解决

有一次在写项目的过程中，使用到的图片原本是png格式的，但是给到我的图片的后缀名被篡改成.jpg的，然后项目一直报这个错。经过一番折腾，最终才发现，原因是图片资源文件的后缀名被篡改，改回png后，项目正常运行。

发现问题的过程：

  • 首先明确是图片的问题
  • 其次锁定了某个文件，原因：该文件图片比较大，但是后缀却是jpg
  • 按常识来看，png是无损压缩，一般文件大小比较大；jpg是有损压缩的，一般文件大小相对较小。
  • 所以果断断定是此文件的问题，然后改了后缀，项目成功跑起来

二、如何不从文件后缀区分JPEG (jpg)、PNG (png)、GIF (gif)等图片文件？


根据文件名后缀只是一个快捷的方法，不用读取文件内容就判断文件类型，但不是唯一的方法

一般文件内容开头都会有一个文件类型的标记：
这是一个PNG文件的内容开头的部分：

可以看到不仅有有文件类型的标记，还有尺寸、色彩类型等等等信息
