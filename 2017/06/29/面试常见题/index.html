3.1 堆

分为两部分：年轻代 和 老年代

Perm 是永久代，不属于堆内存中的一部分，同时 jdk1.8 之后 永久代 也将被移除。

年轻代 –> Minor GC
老年代 –> Full/Major GC

以 hotspot 为例，年轻代中，有 Eden区 和 Suvivor区，比例为8：1。

  • Eden区：新创建的对象，经过一次 Minor GC 后，需要被回收的对象会被放入 Suvivor区。
  • Suvivor区：该区分为 交替的from、to 两个部分（使用复制算法，总有一块为空）
  • 经历多次 Minor GC 的对象，会被从Suvivor区放入老年代
